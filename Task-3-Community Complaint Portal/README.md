# CivicConnect - Community Complaint Portal

CivicConnect is a modern, portfolio-ready full-stack web application that allows citizens to lodge and track community complaints regarding public issues such as garbage collection, road damage, water leaks, streetlights, drainage, and electricity. Built with an Express.js backend and a beautiful glassmorphic Vanilla CSS/JS frontend, the application is powered by Supabase (PostgreSQL) and is fully ready for deployment on Vercel.

---

## 🚀 Features

- **Lodge Complaints**: Citizens can submit complaints with name, email, phone, category, description, location, and priority.
- **Interactive Dashboard**: Dynamically updated KPI cards showcasing:
  - Total Complaints
  - Pending Review
  - In Progress Issues
  - Resolved Issues
  - High Priority Issues
- **Search, Filter & Sort**:
  - Live-search complaints by citizen name, category, or location.
  - Filter complaints instantly by status.
  - Sort complaints by newest or oldest report times.
- **Complete CRUD Operations**: Create new entries, view individual details, edit report properties/status, and delete entries.
- **Polished UI/UX**:
  - Deep blue to violet modern gradient theme.
  - Transparent glassmorphism panels (`backdrop-filter`).
  - Fluid CSS micro-animations on cards, buttons, and loading states.
  - Auto-refreshing list & stats after database modifications.
  - Confirm delete modals and animated toast notifications.
  - Full mobile responsiveness (CSS Grid and Flexbox).

## 🛠️ Technology Stack

**Frontend:**
- HTML5
- CSS3 (Vanilla CSS, custom HSL theme variables, glassmorphic layout)
- Vanilla JavaScript (ES6+, DOM manipulation, Fetch API, debounced search)

**Backend:**
- Node.js & Express.js (REST API, MVC Architecture)
- Express-Validator (Input validation & sanitization)
- Morgan (HTTP request logging)
- CORS & Dotenv (Security & configuration environment variables)

**Database:**
- Supabase (PostgreSQL engine)
- `@supabase/supabase-js` (Database ORM client)

## 🔧 Installation & Local Setup

### 1. Clone or Download the Repository
Navigate to the directory and ensure you are in the project folder:
```bash
cd Community-Complaint-Portal
```

### 2. Install Project Dependencies
Use `npm` to install all package modules:
```bash
npm install
```

### 3. Setup Your Environment Variables
Create a `.env` file in the root directory (based on `.env.example`):
```ini
PORT=5000
SUPABASE_URL=https://your-project-reference.supabase.co
SUPABASE_KEY=your-supabase-api-key
```
> **Note**: For `SUPABASE_KEY`, you can use the `anon` key if your table has open permissions, or the `service_role` key (recommended for backend-only access) which bypasses Row Level Security.

### 4. Supabase Database Table Setup
Log into your [Supabase Dashboard](https://supabase.com), go to the **SQL Editor**, and execute the following query to create the `complaints` table:

```sql
-- Create complaints table
CREATE TABLE complaints (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT NOT NULL,
  category TEXT NOT NULL,
  description TEXT NOT NULL,
  location TEXT NOT NULL,
  priority TEXT DEFAULT 'Medium' NOT NULL CHECK (priority IN ('Low', 'Medium', 'High')),
  status TEXT DEFAULT 'Pending' NOT NULL CHECK (status IN ('Pending', 'In Progress', 'Resolved')),
  created_at TIMESTAMPTZ DEFAULT now() NOT NULL
);

-- (Optional) If you are using the anon public key and want to bypass RLS, disable it:
ALTER TABLE complaints DISABLE ROW LEVEL SECURITY;
```

### 5. Run the Application
Start the development server with hot-reloading (via nodemon):
```bash
npm run dev
```
Open your browser and navigate to: `http://localhost:5000`

## 📡 API Endpoints

All routes expect and return JSON payloads.

| Method | Endpoint | Description | Request Body Rules |
| :--- | :--- | :--- | :--- |
| **GET** | `/api/complaints` | Retrieve all complaints (supports `search`, `status` filters and `sort` parameters) | None |
| **GET** | `/api/complaints/:id` | Fetch details of a single complaint | None |
| **POST** | `/api/complaints` | Log a new complaint | Validates email, numeric phone (10-15 digits), min 10 chars description |
| **PUT** | `/api/complaints/:id` | Edit details/status of an existing complaint | Validates modified fields |
| **DELETE** | `/api/complaints/:id` | Permanently remove a complaint | None |

## ☁️ Vercel Deployment

This project is fully ready to deploy to **Vercel** as an Express API:

1. Push the code to a Git repository (GitHub/GitLab).
2. Link the repository to your Vercel Account.
3. In the project settings on Vercel, add the Environment Variables:
   - `SUPABASE_URL`
   - `SUPABASE_KEY`
4. Deploy. Vercel will automatically read `vercel.json` and host the serverless functions routing static files from the `public` directory.
